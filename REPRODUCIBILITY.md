# Reproducibility Guide — MLCheM Selector

This document explains how to reproduce the machine-learning results reported in
the manuscript (Section 6.2: dataset, model selection, held-out evaluation,
Table 11, and the confusion matrix). It addresses reviewer requests for a
reproducible dataset, training script, split, environment, and license.

---

## 2. The dataset (important)

`Dataset.xlsx` contains **real records extracted from published computational-
chemistry studies**, not synthetically generated data. Each row corresponds to a
reported calculation with the following columns:

| Column | Meaning |
|---|---|
| `Paper` | Source publication identifier (used to prevent train/test leakage) |
| `Primary Method` | The main computational method reported in that study (the label source) |
| `property` | Property category |
| `sub_property` | Specific sub-property |
| `Application Domain` | Application context |
| `System Type` | Chemical/material system studied |

**Labelling rule (not synthetic):** the target label is the *primary method
reported in the paper*, canonicalised into three classes — `COSMO-RS`, `DFT`,
`MD` (e.g., AIMD / NEMD / FPMD → MD). Records outside these three classes were
too sparse and were excluded. The exact rule is in `canonicalize_method()` in
`train.py`.

> Place `Dataset.xlsx` inside the `Model/` folder before running `train.py`.

## 3. Reproduce the results

```bash
cd Model
pip install -r requirements.txt
python train.py
```

This regenerates, next to the script:

- `production_chemistry_classifier.pkl` — the trained pipeline + label encoder
- `classification_report.txt` — per-class precision/recall/F1 (Table 11)
- `confusion_matrix.png` — the held-out confusion matrix (Figure)
- `metrics_summary.json` — accuracy, macro-F1, baseline, and lift

**Expected results** (fixed seed `RANDOM_STATE = 42`, leak-proof paper-grouped split):

| Metric | Value |
|---|---|
| Records (after cleaning) | 323 |
| Class distribution | DFT 143 · MD 103 · COSMO-RS 77 |
| Train / Test split | 273 rows (266 papers) / 50 rows (48 papers) |
| Selected model | Complement Naïve-Bayes (α = 0.5) |
| Held-out accuracy | 0.880 |
| Held-out macro-F1 | 0.877 |
| Zero-R baseline | 0.443 |
| Improvement over baseline | +43.7 points |

## 4. Methodology summary (for reviewers)

1. **Features:** four text fields concatenated → TF-IDF (unigrams+bigrams, 1500 terms, sublinear TF, English stop-words).
2. **Task:** single-label classification (COSMO-RS / DFT / MD).
3. **Leak-proof split:** `GroupShuffleSplit` keyed on `Paper`; the TF-IDF vectoriser is fitted on the training pool only.
4. **Model selection:** 5-fold `StratifiedGroupKFold` cross-validation over nine candidates; the model within 0.5% of the best CV macro-F1 with the **smallest train–CV generalisation gap** is selected.
5. **Evaluation:** classification metrics + confusion matrix on the blind held-out set; comparison against a Zero-R majority baseline.

## 5. Optional AI-validation component

The web interface also shows an **optional** second opinion from a Groq-hosted
LLM (`openai/gpt-oss-120b`, temperature 0.1, JSON output, prompt-grounded on the
same method profiles). It is an interpretability aid only and **does not affect**
the classifier's recommendation; it is excluded from the quantitative evaluation
above because LLM outputs are non-deterministic. See `backend/app.py`.

## 6. Archival with a citable DOI (Zenodo)

To provide a versioned, citable archive:

1. Push the repository to GitHub (dataset, `train.py`, notebook, model, `requirements.txt`, `LICENSE`, this guide).
2. Create a GitHub **Release** (e.g., tag `v1.0.0`).
3. Log in to [zenodo.org](https://zenodo.org) with GitHub, enable the repository under **Settings → GitHub**, then publish the release — Zenodo mints a DOI automatically.
4. Add the DOI badge/citation to the top-level `README.md` and cite it in the manuscript's data-availability statement.
